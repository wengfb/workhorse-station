# 前端与 UI 规则

- 整体风格参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰
- 不使用左侧栏
- 顶栏将聊天 / 概览拆成独立菜单项；项目入口只通过项目下拉进入，不单独展示项目菜单项和 Worktree 按钮
- 顶栏提供项目下拉：展示项目列表，每个项目右侧提供进入项目页的小按钮，列表末尾提供添加项目入口
- 聊天页采用类似 ChatGPT 的结构：左侧独立贴边的聊天会话列表，右侧聊天区占满剩余空间；聊天内容和输入框最大宽度 768px，并在右侧区域居中；聊天会话与 Claude Code 会话必须区分
- 聊天输入框只保留输入、发送和文件选择；生成笔记、待办、提示词等能力作为默认 Skill 提供，不作为主界面操作按钮
- 概览页承载全局笔记、全局 Skill、最近项目、最近会话等管理入口
- 进入具体项目后，使用项目级顶部导航承载总览 / 待办 / 笔记 / Skill / 会话 / Worktree
- 页面不固定包含终端；终端只在选中或创建 Claude Code 会话后出现
- 会话以可关闭的大模态框承载，关闭代表后台运行，不代表停止会话
- PC 版会话模态框采用 3:7 左右布局：左侧会话列表/信息，右侧操作终端或会话历史
- 移动版采用单列和全屏弹层，终端/历史通过 tab 切换，不做多栏挤压
- 列表页优先支持筛选、搜索、排序、状态标签
- 详情页优先支持属性查看、关联关系、草稿确认；编辑、新建和删除确认优先使用 modal 或移动端 drawer
- 深色优先，高对比、少动效、强调状态
- 需要终端执行和其他事项并行时，优先让会话后台运行，主界面继续处理笔记、待办、Skill 或项目管理

**Why:** 这个产品更像开发工作台，不是传统后台；会话终端需要足够宽，同时不能阻塞用户继续整理事项。
**How to apply:** 所有页面设计都先问自己是否区分了全局工作台、项目工作台和会话执行上下文。
