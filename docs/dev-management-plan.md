# 开发计划：AI 开发管理 Web 软件

## 1. 项目目标
做一个单用户的开发管理工作台，围绕 **项目 / worktree / 笔记 / 待办 / Skill / AI 聊天 / Claude Code 会话** 形成闭环。

## 2. 参考项目结论
参考 Codeman 后，建议复用它的这些能力：
- PTY 会话管理
- SSE 实时推送
- 持久化状态存储
- 任务队列与 prompt 执行模型
- 终端交互和会话恢复能力

需要重做或新增的能力：
- 项目与 worktree 一等公民
- 笔记系统
- 待办系统
- Skill 管理
- 首页式通用 AI 聊天入口
- AI 驱动的待办 / prompt 生成
- 可后台运行的 Claude Code 会话模态框
- 文件处理标准 Skill（Excel / Word / PDF / 图片的导入、抽取、搜索、OCR、摘要）

## 3. MVP 范围
### 必做
- 项目管理
- worktree 管理
- 笔记管理（全局 / 项目）
- 待办管理（项目内）
- Skill 文件夹管理（全局 `~/.claude/skills/*` / 项目 `.claude/skills/*`）
- 首页通用 AI 聊天入口
- Claude Code 会话接入
- prompt 生成与编辑
- 会话模态框：PC 3:7，移动端全屏
- 会话创建：直接创建 / 从待办创建

### 暂不做
- 多用户
- 权限系统
- 团队协作
- 复杂工作流引擎
- 代码语义索引
- 多模型调度
- 自主代理执行

## 4. 技术路线
### 风格原则
- 整体视觉与交互参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰
- 不使用左侧栏
- 顶栏将聊天 / 概览拆成独立菜单项；项目入口只通过项目下拉进入，不单独展示项目菜单项和 Worktree 按钮
- 顶栏提供项目下拉：展示项目列表，每个项目右侧提供进入项目页的小按钮，列表末尾提供添加项目入口
- 聊天页采用类似 ChatGPT 的结构：左侧聊天会话列表，右侧聊天内容和输入框；聊天会话与 Claude Code 会话区分管理
- 聊天输入框支持选择文件；生成笔记、待办、提示词等能力作为默认 Skill，不作为主界面按钮
- 概览页承载全局笔记、全局 Skill、最近项目、最近聊天、运行中 Claude Code 会话等管理入口
- 进入具体项目后，使用项目级顶部导航承载总览 / 待办 / 笔记 / Skill / 会话 / Worktree
- 终端只属于会话执行上下文，不做全局固定右侧栏
- 以 Codeman 的服务结构为参考
- 提供 REST API + SSE，终端交互使用 WebSocket
- 统一封装会话、笔记、待办、Skill、AI 服务
- 集中式存储（建议 SQLite 或轻量 JSON 起步，后续可切数据库）

### 前端
- 首页保留现代 AI 聊天工具式布局：左侧聊天会话列表，右侧聊天内容和输入框
- 首页概览承载全局笔记、全局 Skill、最近项目、最近聊天和运行中 Claude Code 会话入口
- 项目内对象管理优先采用列表 + 详情，移动端采用单列 drill down
- 会话通过大模态框打开，关闭后后台运行
- PC 会话模态框使用 3:7 左右布局；移动端全屏

### AI 层
- 统一 LLM Service
- 聊天智能体采用 Claude SDK 作为主入口：Claude 负责聊天、流式输出、工具调用；程序层提供 skill、文件处理、待办生成、prompt 生成等工具供其调用；如需更细的参数校验，使用 Zod
- 支持：
  - 生成待办草稿
  - 生成 Claude Code prompt
  - 生成笔记摘要
  - 聊天式交互
  - 直接创建或从待办创建 Claude Code 会话
- 所有 AI 输出先生成草稿，再由用户确认落库

## 5. 开发阶段

- 首页 AI 聊天、全局笔记、完整 Skill 系统仍然属于 MVP，但放在主闭环跑通之后补齐。

## 7. 关键交互主线
### 当前优先主线（项目内闭环）
1. 进入项目并选择 worktree（worktree 隶属于项目，存放在项目 `.claude/worktree/` 下）
2. 记录项目笔记或整理已有笔记
3. 从笔记生成待办，或直接创建项目待办
4. 用户确认待办内容、状态、标签和来源关联
5. 待办生成 prompt 草稿
6. 用户修改并保存 prompt 草稿
7. 发起 Claude Code 会话记录（直接创建或从待办创建，可关联已有 worktree 或填写期望名称）
8. 在会话模态框中查看和切换已保存会话
9. 收起会话窗口后台运行
10. 终端执行与停止控制已接入，后续继续补齐更完整的回放与恢复体验

### 后续补齐主线（首页与全局能力）
1. 在首页打开聊天，或切换到概览查看全局管理入口
2. 通过默认 Skill 生成笔记、待办或 prompt 草稿
3. 用户确认后保存到全局或项目上下文
4. 需要执行代码任务时，进入项目 Claude Code 会话流程

## 8. 风险点
- worktree 与会话的绑定关系要清晰，且 worktree 归属到具体项目
- 删除 worktree 时要同步清理对应的本地 git 分支
- AI 输出必须可编辑，不能直接写入
- 存储设计要能支持后续扩展
- 会话后台运行状态要清晰，避免用户误以为关闭窗口就是停止会话
- 移动端不能沿用桌面多栏布局
- 需要避免把功能做成另一个复杂的 agent 控制台
- 如果先做大量首页能力而不接真实会话，容易出现“看起来完整但主闭环不可用”的偏差

## 9. 第一版验收标准
### 最小可用闭环
- 能创建项目并绑定目录
- 能管理 worktree
- 进入项目后可通过顶部导航切换项目总览、待办、笔记、Skill、会话、Worktree
- 能写项目笔记并转待办
- 能从待办生成并编辑 prompt
- 能直接创建 Claude Code 会话记录或从待办创建会话记录
- 会话创建时可关联已有 worktree，并可填写期望的新 worktree 名称
- 会话通过可关闭的大模态框打开，PC 约 3:7，移动端全屏
- 能查看、重新打开、停止、删除单条已保存会话记录
- 会话记录保留与项目 / 待办 / worktree / prompt draft 的关联

说明：这里的“最小可用闭环”当前已经包括真实 Claude Code PTY、终端交互、停止控制，以及会话结果保存并显式回写到任务 / 项目；后续再继续补齐更完整的历史回放与结果留存体验。

### MVP 补齐项
- 首页可通过顶栏独立菜单进入聊天 / 概览，通过项目下拉进入项目
- 聊天页区分聊天会话和 Claude Code 会话，输入框支持文件选择
- 能通过 AI 生成待办和 prompt
- 支持全局笔记、Skill 和概览入口
