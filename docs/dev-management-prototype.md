# 开发管理 Web 软件原型

## 1. 设计目标
- 单用户
- 支持 PC 和移动版
- 单代码目录 + 项目级 worktree
- 无左侧栏
- 顶栏将聊天 / 概览拆成独立菜单项，不再使用模式下拉；项目入口只通过项目下拉进入，且不单独展示 Worktree 按钮
- 聊天页采用类似 ChatGPT 的结构：左侧聊天会话列表，右侧聊天内容和输入框
- 聊天会话与 Claude Code 会话区分管理；Claude Code 会话仍在项目会话区打开
- 聊天输入框支持文件选择；生成笔记、待办、提示词等能力作为默认 Skill，不作为聊天主界面按钮
- 进入具体项目后显示项目级顶部导航：总览 / 待办 / 笔记 / Skill / 会话 / Worktree
- 终端只属于 Claude Code 会话，不作为全局固定右侧栏
- 选中或创建会话后打开可关闭的大模态框；PC 内部 3:7，移动端全屏
- 整体风格参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰
- AI 聊天底层采用 Anthropic SDK + Zod + 轻量编排，前端可选 Vercel AI SDK

│                                                                              │

## 3. 页面原型

### 3.1 首页：聊天
用途：提供类似 ChatGPT 的轻量聊天入口。聊天会话只记录 AI 聊天上下文，不等同于 Claude Code 会话。

```text
[顶栏菜单：聊天]

┌ 聊天会话列表 ───────────┬ 聊天区 ──────────────────────────────┐
│ 新建聊天                │ 用户 / AI 消息                         │
│ chat-1                  │                                      │
│ chat-2                  │ [选择文件] [输入消息........] [发送]   │
└────────────────────────┴──────────────────────────────────────┘
```

约定：
- 聊天区只显示消息和输入框，避免堆叠大量动作按钮
- 输入框支持选择文件
- 生成笔记、待办、提示词等能力作为默认 Skill，由聊天自然语言触发
- 直接创建 Claude Code 会话仍进入项目会话流程

### 3.2 首页：概览
用途：承载全局管理入口和最近状态。

```text
[顶栏菜单：概览]

[全局笔记] [全局 Skill] [最近项目] [最近聊天] [运行中 Claude Code 会话]
```

概览内容：
- 全局笔记管理入口
- 全局 Skill 管理入口
- 最近项目 / worktree 状态
- 最近聊天会话
- 运行中的 Claude Code 会话入口

### 3.3 项目总览
用途：快速查看项目状态。

内容：
- 项目名称、路径、默认分支
- 当前 worktree 状态
- dirty / clean
- 最近笔记
- 未完成待办
- 最近 Claude Code 会话
- 快捷动作：写笔记、创建待办、发起会话、创建 worktree

### 3.4 项目待办
用途：管理项目内开发事项。

```text
PC：待办列表 + 详情
移动：待办列表 -> 待办详情
```

待办详情：
- 标题、描述、状态、标签
- 来源笔记
- 生成 prompt
- 从待办创建 Claude Code 会话

### 3.5 项目笔记
用途：管理项目相关笔记，并可转待办。

```text
PC：笔记列表 + 详情
移动：笔记列表 -> 笔记详情
```

### 3.6 项目 Skill
用途：管理项目级 Skill，可覆盖全局 Skill。

内容：
- 名称
- 作用范围：项目
- 提示词
- 脚本
- 参数说明
- 在 prompt / 会话中注入

### 3.7 项目 Worktree
用途：管理代码目录和项目级 worktree。

内容：
- worktree 列表
- 创建 worktree
- 切换当前 worktree
- 删除 worktree 时同步删除对应本地 git 分支

### 3.8 项目会话
用途：查看、创建、恢复和管理 Claude Code 会话。

```text
┌ 会话列表 ─────────────────────────────────────────────┐
│ session-1  [draft]   [项目: A] [待办: bugfix] [停止] [删除] │
│ session-2  [completed] [Worktree: feat-x]            │
└──────────────────────────────────────────────────────┘

操作：
- 直接创建会话
- 从待办创建会话
- 通过顶部“会话”按钮随时打开会话模态框
- 打开会话执行模态框
```

当前交互约定：
- 创建流程先通过独立表单模态框生成 / 保存 prompt 草稿，再创建会话记录
- 会话执行模态框只负责查看和切换已创建会话
- 左侧列表使用紧凑标签展示项目、待办、worktree、状态等基础信息
- 每条会话在列表右侧提供单独的停止、删除操作
- 右上角只保留关闭按钮；关闭仅表示收起窗口，不表示停止会话
- 右侧执行区当前仍是 PTY 未接入占位，后续再接真实 Claude Code 终端和历史

## 4. 关键交互流程

### 流程 A：笔记 → 待办 → 会话
1. 新建笔记
2. AI 生成待办草稿
3. 用户确认待办
4. 待办生成 prompt 草稿
5. 用户修改 prompt
6. 选择或创建 worktree
7. 从待办创建 Claude Code 会话
8. 打开会话模态框操作终端或收起后台运行

### 流程 B：首页聊天驱动
1. 打开首页聊天
2. 选择聊天会话，或新建聊天会话
3. 在输入框中输入消息，可选择文件作为上下文
4. 通过默认 Skill 生成笔记、待办或 prompt 草稿
5. 用户确认后保存到概览或项目上下文
6. 如需执行代码任务，再进入项目会话流程创建 Claude Code 会话

### 流程 C：会话后台运行
1. 在项目会话页选择已有会话，或先打开创建会话表单模态框
2. 生成并确认 prompt 草稿，创建会话记录
3. 打开会话执行模态框
4. 关闭模态框，回到主界面继续处理待办、笔记、Skill
5. 通过顶部“会话”按钮或项目会话页重新打开模态框
6. 如需结束某条会话，在左侧会话列表中单独点击“停止”

## 5. PC / 移动响应式

### PC
- 首页主内容居中，列表管理页可用列表 + 详情
- 项目内对象管理页优先使用列表 + 详情
- 会话模态框宽屏显示，内部 3:7，终端侧至少占主要宽度

### 移动
- 不使用左右分栏
- 顶部导航允许横向滚动或折叠
- 列表点击进入详情
- 新建、编辑、删除确认使用全屏弹层或 drawer
- 会话模态框全屏，终端 / 历史通过 tab 切换

## 6. 设计原则
- 所有 AI 输出先作为草稿
- 所有核心对象都可回溯来源
- 项目、worktree、笔记、待办、Skill、会话要互相关联
- worktree 只属于项目，存放在项目 `.claude/worktree/` 下，删除时同步删除本地 git 分支
- 终端是会话执行上下文，不是全局页面布局的一部分
- 关闭会话窗口只表示后台运行，不代表停止
- 少导航，但要区分全局工作台、项目工作台、会话执行上下文

## 7. 第一版视觉重点
- 首页像现代 AI 在线聊天工具：左侧聊天会话列表，右侧简洁聊天区
- 左上角菜单切换聊天 / 概览，项目通过项目下拉进入
- 概览集中呈现全局笔记、全局 Skill、最近项目、最近聊天和运行中 Claude Code 会话入口
- 项目内顶部导航清晰表达当前项目上下文
- 会话终端足够宽，可随时收起后台运行
- 移动端保持单列和全屏操作，不挤压多栏
