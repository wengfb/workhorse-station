# 开发管理 Web 软件 - MVP 需求说明

## 1. 项目目标
基于 Codeman 的思路，开发一个面向单用户的开发管理 Web 软件，核心目标是把 **项目、worktree、笔记、待办、Skill、AI 聊天、Claude Code 会话** 串成一个闭环，提升开发任务整理和执行效率。

## 2. 核心定位
- 单用户使用
- 一个项目绑定一个代码目录
- 需要支持项目级 worktree 管理
- 基于 Codeman 现有的 PTY 能力
- 偏集中式存储
- 支持接入大模型
- 聊天智能体基于 Claude SDK：Claude 负责聊天、流式输出、工具调用；程序层提供 skill、文件处理、待办生成、prompt 生成等工具供其调用；如需参数校验，使用 Zod
- 支持 Claude Code，后续可扩展 Codex
- 支持 PC 和移动版

## 3. 核心功能

### 3.1 项目管理
- 创建、编辑、删除项目
- 一个项目绑定一个代码目录
- 记录项目基础信息
  - 名称
  - 路径
  - 默认分支
  - 备注说明
- 显示项目状态
  - 当前 worktree
  - 是否有未提交变更
  - 最近会话
  - 未完成待办

### 3.2 Worktree 管理
- 查看项目下所有 worktree
- 显示 worktree 路径、分支、状态
- 选择当前操作 worktree
- 会话可绑定指定 worktree
- worktree 基于项目管理，存放在项目 `.claude/worktree/` 目录下
- 基于任务启动会话时，可填写或选择已有 worktree 名称
- 删除 worktree 时同步删除对应的本地 git 分支

### 3.3 随手记
- 支持全局笔记和项目笔记
- 首页可管理全局笔记
- 进入项目后可管理项目笔记
- 笔记支持：
  - 新建
  - 编辑
  - 删除
  - 命名
  - 标签
- 支持 AI 自动分类，但仅作为标签，不替代人工整理
- 笔记可关联：
  - 项目
  - 待办
  - 会话

### 3.4 待办管理
- 待办属于具体项目
- 支持从笔记生成待办
- 待办支持：
  - 标题
  - 描述
  - 状态
  - 标签
  - 来源笔记关联
- 支持 AI 辅助生成待办草稿
- 用户确认后再保存
- 支持从待办生成 prompt 草稿并创建 Claude Code 会话

### 3.5 Prompt 生成
- 根据待办生成 Claude Code 任务提示词
- 支持人工修改
- 支持保存草稿和历史版本
- 可插入项目上下文、worktree 状态、Skill 文件夹引用

### 3.6 通用 AI 聊天入口
- [x] 首页默认提供类似 ChatGPT 的聊天入口
- [x] 顶栏菜单包含聊天 / 概览，项目入口通过项目下拉进入
- [x] 聊天会话与 Claude Code 会话区分管理
- [x] 聊天页布局：左侧聊天会话列表，右侧聊天内容和输入框
- [x] 聊天输入框支持选择文件
- [x] 生成笔记、待办、prompt 等能力作为默认 Skill，通过自然语言触发，不作为主界面按钮
- [x] 聊天时可自动绑定当前项目 / worktree / 上下文
- [x] AI 输出可转成结构化结果，供用户确认后落库

### 3.7 Skill 管理
- 支持全局 Skill 和项目级 Skill
- 首页可管理全局 Skill 文件夹，来源为 `~/.claude/skills/*`
- 进入项目后可管理项目级 Skill 文件夹，来源为项目 `.claude/skills/*`
- Skill 名称等于文件夹名
- 同名 Skill 在项目下可覆盖全局版本
- Skill 管理只负责发现、创建、重命名、删除和复制整个文件夹，并可直接编辑 `SKILL.md`；除该文档外不解析目录内部其他文件
- AI 聊天和 prompt 生成时支持注入 Skill
- 会话创建时可选择注入 Skill 并记录使用快照
- 文件处理可以作为标准 Skill，通过 AI 聊天调用，支持 Excel / Word / PDF / 图片的导入、抽取、搜索、OCR 和摘要

### 3.8 Claude Code 会话
- 基于现有 PTY 能力发起会话，支持真实 Claude Code 终端交互
- 会话绑定：
  - 项目
  - worktree
  - prompt
  - 关联待办
  - 使用的 Skill 快照
- 支持两种创建方式：
  - 直接创建会话
  - 从待办事项创建会话
- 当前已支持 prompt 草稿预览、编辑、保存后再创建会话记录
- 当前已支持查看、恢复、停止、删除单条已保存会话记录
- 当前已支持在会话历史中保存结果摘要，并显式写回关联待办或当前项目
- 记录会话输出、终端快照和摘要
- 支持从历史会话继续（当前为重新打开已保存记录）
- 选中会话后打开可关闭的大模态框
- 关闭会话模态框表示后台运行，不等于停止会话

## 4. 界面布局
- 首页：顶部导航 + 主内容区，顶栏将聊天 / 概览拆成独立菜单项；项目入口通过项目下拉进入
- 顶栏提供项目下拉：展示项目列表、每个项目的进入按钮，以及末尾添加项目入口
- 聊天页：左侧聊天会话列表，右侧聊天内容和带文件选择的输入框
- 概览页：全局笔记、全局 Skill、最近项目、最近聊天、运行中 Claude Code 会话等管理入口
- 项目内：顶部项目导航 + 主内容区，导航包含总览 / 待办 / 笔记 / Skill / 会话 / Worktree
- 会话执行：通过模态框打开，不作为全局固定右侧栏
- PC 会话模态框：左侧 30% 会话列表/信息，右侧 70% 操作终端或会话历史
- 移动端：单列布局，会话模态框全屏，终端 / 历史通过 tab 切换
- 整体风格参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰

### 交互要求
- 首页默认显示聊天主界面
- 页面不固定展示终端
- 只有选中或创建 Claude Code 会话后才展示终端
- 会话窗口可关闭并后台运行
- 真正停止会话必须使用单独的停止操作和确认
- 编辑、新建、删除确认优先使用模态框或移动端抽屉
- 支持在会话后台运行时继续操作笔记、待办、Skill、项目和 worktree

## 5. MVP 闭环
推荐最小闭环为：

1. 在首页聊天中通过默认 Skill 生成笔记或待办草稿
2. 用户确认后保存到全局或项目上下文
3. 进入项目并选择 worktree
4. 根据待办生成 Claude Code prompt
5. 直接创建会话或从待办创建会话
6. 在会话模态框中操作终端或查看历史
7. 收起会话窗口后台运行
8. 保存会话结果和摘要

## 6. 暂不做
- 多用户权限
- 团队协作
- 复杂工作流引擎
- 代码语义索引
- 多模型统一调度
- 自主代理式执行
