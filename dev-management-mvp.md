# 开发管理 Web 软件 - MVP 需求说明

## 1. 项目目标
基于 Codeman 的思路，开发一个面向单用户的开发管理 Web 软件，核心目标是把 **项目、worktree、笔记、待办、Skill、AI 聊天、Claude Code 会话** 串成一个闭环，提升开发任务整理和执行效率。

## 2. 核心定位
- 单用户使用
- 一个项目绑定一个代码目录
- 需要支持项目级 worktree 管理
- 基于 Codeman 现有的 PTY 能力
- 偏集中式存储
- 支持接入大模型
- 聊天智能体基于 Claude SDK：Claude 负责聊天、流式输出、工具调用；程序层提供 skill、文件处理、待办生成、prompt 生成等工具供其调用；如需参数校验，使用 Zod
- 支持 Claude Code，后续可扩展 Codex

## 3. 核心功能

### 3.1 项目管理
- 创建、编辑、删除项目
- 一个项目绑定一个代码目录
- 记录项目基础信息
  - 名称
  - 路径
  - 默认分支
  - 备注说明
- 显示项目状态
  - 当前 worktree
  - 是否有未提交变更
  - 最近会话
  - 未完成待办

### 3.2 Worktree 管理
- 查看项目下所有 worktree
- 显示 worktree 路径、分支、状态
- 选择当前操作 worktree
- 会话可绑定指定 worktree
- worktree 基于项目管理，存放在项目 `.claude/worktree/` 目录下
- 基于任务启动会话时，可填写或选择已有 worktree 名称
- 删除 worktree 时同步删除对应的本地 git 分支

### 3.3 随手记
- 支持全局笔记和项目笔记
- 笔记支持：
  - 新建
  - 编辑
  - 删除
  - 命名
  - 标签
- 支持 AI 自动分类，但仅作为标签，不替代人工整理
- 笔记可关联：
  - 项目
  - 待办
  - 会话

### 3.4 待办管理
- 支持从笔记生成待办
- 待办支持：
  - 标题
  - 描述
  - 状态
  - 标签
  - 来源笔记关联
- 支持 AI 辅助生成待办草稿
- 用户确认后再保存

### 3.5 Prompt 生成
- 根据待办生成 Claude Code 任务提示词
- 支持人工修改
- 支持保存草稿和历史版本
- 可插入项目上下文、worktree 状态、Skill 内容

### 3.6 通用 AI 聊天窗口
- 提供统一的 AI 交互入口
- 支持：
  - 生成笔记
  - 生成待办
  - 生成 prompt
  - 发起 Claude Code 会话
- 聊天时可自动绑定当前项目 / worktree / 上下文
- AI 输出可转成结构化结果，供用户确认后落库

### 3.7 Skill 管理
- 支持全局 Skill 和项目级 Skill
- Skill 内容包含：
  - 提示词
  - 脚本
  - 参数说明
- 同名 Skill 在项目下可覆盖全局版本
- AI 聊天和 prompt 生成时支持注入 Skill
- 文件处理可以作为标准 Skill，通过 AI 聊天调用，支持 Excel / Word / PDF / 图片的导入、抽取、搜索、OCR 和摘要

### 3.8 Claude Code 会话
- 基于现有 PTY 能力发起会话
- 会话绑定：
  - 项目
  - worktree
  - prompt
  - 关联待办
- 记录会话输出和摘要
- 支持从历史会话继续

## 4. 界面布局
- 顶栏：项目切换、全局搜索、常用操作
- 中间主内容区：笔记 / 待办 / Skill / 会话列表与详情
- 右侧面板：终端 / AI 聊天 / 会话输出，可切换、可全屏
- 整体风格参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰

### 交互要求
- 默认右侧显示终端
- 支持全屏终端
- 支持在主内容区继续操作笔记、待办等功能
- 终端与内容区可并行使用

## 5. MVP 闭环
推荐最小闭环为：

1. 选择项目和 worktree  
2. 使用 AI 聊天生成笔记或待办  
3. 根据待办生成 Claude Code prompt  
4. 通过 PTY 发起会话  
5. 保存会话结果和摘要  

## 6. 暂不做
- 多用户权限
- 团队协作
- 复杂工作流引擎
- 代码语义索引
- 多模型统一调度
- 自主代理式执行
