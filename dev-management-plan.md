# 开发计划：AI 开发管理 Web 软件

## 1. 项目目标
做一个单用户的开发管理工作台，围绕 **项目 / worktree / 笔记 / 待办 / Skill / AI 聊天 / Claude Code 会话** 形成闭环。

## 2. 参考项目结论
参考 Codeman 后，建议复用它的这些能力：
- PTY 会话管理
- SSE 实时推送
- 持久化状态存储
- 任务队列与 prompt 执行模型
- 终端交互和会话恢复能力

需要重做或新增的能力：
- 项目与 worktree 一等公民
- 笔记系统
- 待办系统
- Skill 管理
- 首页式通用 AI 聊天入口
- AI 驱动的待办 / prompt 生成
- 可后台运行的 Claude Code 会话模态框
- 文件处理标准 Skill（Excel / Word / PDF / 图片的导入、抽取、搜索、OCR、摘要）

## 3. MVP 范围
### 必做
- 项目管理
- worktree 管理
- 笔记管理（全局 / 项目）
- 待办管理（项目内）
- Skill 管理（全局 / 项目）
- 首页通用 AI 聊天入口
- Claude Code 会话接入
- prompt 生成与编辑
- 会话模态框：PC 3:7，移动端全屏
- 会话创建：直接创建 / 从待办创建

### 暂不做
- 多用户
- 权限系统
- 团队协作
- 复杂工作流引擎
- 代码语义索引
- 多模型调度
- 自主代理执行

## 4. 技术路线
### 风格原则
- 整体视觉与交互参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰
- 不使用左侧栏
- 首页作为全局工作台，左上角下拉仅切换聊天 / 概览
- 聊天页采用类似 ChatGPT 的结构：左侧聊天会话列表，右侧聊天内容和输入框；聊天会话与 Claude Code 会话区分管理
- 聊天输入框支持选择文件；生成笔记、待办、提示词等能力作为默认 Skill，不作为主界面按钮
- 概览页承载全局笔记、全局 Skill、最近项目、最近聊天、运行中 Claude Code 会话等管理入口
- 进入具体项目后，使用项目级顶部导航承载总览 / 待办 / 笔记 / Skill / 会话 / Worktree
- 终端只属于会话执行上下文，不做全局固定右侧栏
- 以 Codeman 的服务结构为参考
- 提供 REST API + SSE，终端交互使用 WebSocket
- 统一封装会话、笔记、待办、Skill、AI 服务
- 集中式存储（建议 SQLite 或轻量 JSON 起步，后续可切数据库）

### 前端
- 首页保留现代 AI 聊天工具式布局：左侧聊天会话列表，右侧聊天内容和输入框
- 首页概览承载全局笔记、全局 Skill、最近项目、最近聊天和运行中 Claude Code 会话入口
- 项目内对象管理优先采用列表 + 详情，移动端采用单列 drill down
- 会话通过大模态框打开，关闭后后台运行
- PC 会话模态框使用 3:7 左右布局；移动端全屏

### AI 层
- 统一 LLM Service
- 聊天智能体采用 Claude SDK 作为主入口：Claude 负责聊天、流式输出、工具调用；程序层提供 skill、文件处理、待办生成、prompt 生成等工具供其调用；如需更细的参数校验，使用 Zod
- 支持：
  - 生成待办草稿
  - 生成 Claude Code prompt
  - 生成笔记摘要
  - 聊天式交互
  - 直接创建或从待办创建 Claude Code 会话
- 所有 AI 输出先生成草稿，再由用户确认落库

## 5. 开发阶段

### Phase 0：项目骨架
- [x] 初始化项目结构（pnpm monorepo，`apps/web`、`apps/api`、`packages/shared`）
- [x] 确定前后端目录划分
- [x] 接入基础配置和环境变量
- [x] 建立统一存储层（当前使用 `sql.js` 初始化 `data/app.db`，FTS5 后续再评估）
- [x] 建立日志与错误处理
- [x] 完成首个浏览器可验证工作台骨架
- [x] 验证记录见 `docs/dev-progress.md`

### Phase 1：项目与 worktree
- 项目 CRUD
- 代码目录绑定
- 项目级 worktree 管理（每个项目的 `.claude/worktree/` 下按名称保存）
- worktree 列表、选择、状态展示
- 基于任务启动会话时，可填写或选择已有 worktree 名称
- 删除 worktree 时同步删除对应的本地 git 分支
- 项目状态总览

### Phase 2：笔记与待办
- 全局笔记与项目笔记 CRUD
- 笔记标签与搜索
- 笔记转待办
- 项目内待办状态、标签、来源关联
- 从待办生成 prompt 草稿

### Phase 3：Skill 系统
- 全局 Skill 管理
- 项目级 Skill 覆盖
- Skill 内容支持提示词 + 脚本 + 参数
- 文件处理标准 Skill（Excel / Word / PDF / 图片的导入、抽取、搜索、OCR、摘要）
- 在 prompt 生成和会话创建中注入 Skill

### Phase 4：AI 聊天入口
- 首页通用聊天入口
- 聊天会话与 Claude Code 会话区分管理
- 支持左侧聊天会话列表和右侧简洁聊天区
- 输入框支持选择文件
- 支持项目 / worktree 上下文注入
- 生成笔记 / 待办 / prompt 作为默认 Skill 能力
- 支持调用标准 Skill（包括文件处理 Skill）
- 需要执行代码任务时，进入项目 Claude Code 会话流程

### Phase 5：会话与终端
- 复用 PTY 会话能力
- 会话绑定项目 / worktree / prompt / 待办 / Skill 快照
- 支持直接创建会话和从待办创建会话
- 保存会话记录与摘要
- 会话模态框支持操作终端 / 查看历史
- 关闭会话模态框后会话后台运行

### Phase 6：体验打磨
- 全局搜索
- 快速切换项目 / worktree
- 草稿历史
- 最近会话
- 运行中会话提醒
- 状态总览面板
- 移动端交互优化

## 6. 关键数据模型
- Project
- Worktree
- Note
- Todo
- Skill
- PromptDraft
- AIChatSession
- ClaudeSession
- SessionLog

## 7. 关键交互主线
1. 在首页打开聊天，或切换到概览查看全局管理入口
2. 选择项目 / worktree（worktree 隶属于项目，存放在项目 `.claude/worktree/` 下）
3. 记录笔记或让 AI 生成待办草稿
4. 用户确认待办
5. 待办生成 prompt 草稿
6. 用户修改 prompt
7. 发起 Claude Code 会话（直接创建或从待办创建，可填写或选择已有 worktree 名称）
8. 在会话模态框中操作终端或查看历史
9. 收起会话窗口后台运行
10. 保存会话结果与关联数据

## 8. 风险点
- worktree 与会话的绑定关系要清晰，且 worktree 归属到具体项目
- 删除 worktree 时要同步清理对应的本地 git 分支
- AI 输出必须可编辑，不能直接写入
- 存储设计要能支持后续扩展
- 会话后台运行状态要清晰，避免用户误以为关闭窗口就是停止会话
- 移动端不能沿用桌面多栏布局
- 需要避免把功能做成另一个复杂的 agent 控制台

## 9. 第一版验收标准
- 能创建项目并绑定目录
- 能管理 worktree
- 首页可在聊天 / 概览间切换
- 聊天页区分聊天会话和 Claude Code 会话，输入框支持文件选择
- 进入项目后可通过顶部导航切换项目总览、待办、笔记、Skill、会话、Worktree
- 能写笔记并转待办
- 能通过 AI 生成待办和 prompt
- 能直接创建 Claude Code 会话或从待办创建会话
- 会话通过可关闭的大模态框打开，PC 约 3:7，移动端全屏
- 能保存会话结果
