# 开发管理 Web 软件原型

## 1. 设计目标
- 单用户
- 单代码目录 + 项目级 worktree
- 无左侧栏
- 顶栏 + 中间内容区 + 右侧工作区
- 右侧默认终端，可切换 AI 聊天 / 会话输出
- 整体风格参考 Notion 的数据库页：列表/详情联动、属性化信息展示、清晰分区、低干扰
- AI 聊天底层采用 Anthropic SDK + Zod + 轻量编排，前端可选 Vercel AI SDK

## 3. 页面原型

### 3.1 总览页
用途：快速查看项目状态。

```text
[项目卡片] [当前 worktree] [最近笔记] [未完成待办] [最近会话]
```

内容：
- 项目名称、路径、默认分支
- 当前 worktree 状态
- dirty / clean
- 最近 5 条笔记
- 待办数量统计
- 最近 3 次 Claude Code 会话

### 3.2 项目页
用途：管理代码目录和 worktree。

```text
┌ 项目列表 ────────────────┐
│ project-a                │
│ project-b                │
│ project-c                │
└───────────────────────────┘

右侧：项目详情
- 名称
- 目录
- 默认分支
- worktree 列表
- 创建 worktree 按钮（或基于任务启动会话时填写名称自动创建）
- 切换当前 worktree
- 删除 worktree 时同步删除对应本地 git 分支
```

### 3.3 笔记页
用途：记录随手记，并可转待办。

```text
┌ 笔记列表 ───────────────────────────────┐
│ [标签] [项目] [时间] [标题]             │
│ note-1                                 │
│ note-2                                 │
└────────────────────────────────────────┘

右侧详情：
- 标题
- 内容
- 标签
- 关联项目
- 生成待办按钮
- 删除 / 重命名
```

### 3.4 待办页
用途：管理开发事项。

```text
┌ 待办列表 ───────────────────────────────┐
│ 状态 | 优先级 | 标题 | 来源笔记         │
│ todo-1                               │
│ todo-2                               │
└────────────────────────────────────────┘

右侧详情：
- 标题
- 描述
- 状态
- 标签
- 来源笔记
- 生成 prompt 按钮
- 发起会话按钮
```

### 3.5 Skill 页
用途：管理全局 / 项目级 Skill。

```text
┌ Skill 列表 ────────────────────────────┐
│ global/refactor                       │
│ project-a/debug                       │
│ project-b/review                      │
└────────────────────────────────────────┘

右侧详情：
- 名称
- 作用范围：全局 / 项目
- 提示词
- 脚本
- 参数说明
- 在聊天中注入按钮
- 文件处理 skill：支持 Excel / Word / PDF / 图片导入、抽取、搜索、OCR、摘要
```

### 3.6 AI 聊天页
用途：统一入口生成笔记、待办、prompt。

```text
┌ 聊天区 ────────────────────────────────┐
│ 用户消息                               │
│ AI 回复                                │
│                                        │
│ [输入框：可选择项目 / worktree / skill] │
│ [发送] [生成笔记] [生成待办] [生成 prompt] │
└────────────────────────────────────────┘
```

AI 输出建议支持三种结构化结果：
- 笔记草稿
- 待办草稿
- prompt 草稿

### 3.7 会话页
用途：查看 Claude Code 会话与结果。

```text
┌ 会话列表 ───────────────────────────────┐
│ session-1  running                     │
│ session-2  completed                   │
└────────────────────────────────────────┘

右侧详情：
- 绑定项目 / worktree
- prompt 内容
- 终端输出
- 会话摘要
- 重新发起 / 继续会话
```

## 4. 关键交互流程

### 流程 A：笔记 → 待办 → 会话
1. 新建笔记
2. AI 生成待办草稿
3. 用户确认待办
4. 待办生成 prompt 草稿
5. 用户修改 prompt
6. 发起 Claude Code 会话

### 流程 B：聊天驱动
1. 打开 AI 聊天
2. 选择项目 / worktree / skill
3. AI 生成笔记或待办
4. 一键转入待办 / prompt / 会话

### 流程 C：worktree 驱动
1. 在项目页选择或输入 worktree 名称
2. 若名称不存在，则在该项目的 `.claude/worktree/` 下创建对应目录并关联本地分支
3. 查看 dirty 状态
4. 一键发起会话
5. 会话结果回写到当前项目

## 5. 右侧工作区切换
右侧默认展示终端，支持：
- 终端
- AI 聊天
- 会话输出
- 全屏模式

建议交互：
- 顶栏按钮快速切换
- `Ctrl + Enter` 发起会话
- `Esc` 退出全屏

## 6. 设计原则
- 所有 AI 输出先作为草稿
- 所有核心对象都可回溯来源
- 项目、worktree、笔记、待办、Skill、会话要互相关联
- worktree 只属于项目，存放在项目 `.claude/worktree/` 下，删除时同步删除本地 git 分支
- 少导航，多上下文

## 7. 第一版视觉重点
- 顶栏清晰可切换项目
- 中间区域专注当前对象
- 右侧永远保留“执行区”
- 不使用左侧栏
